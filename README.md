# Rufen × Cara8 — Outbound Autopilot

**AI calling campaigns for car dealerships.** Upload a list of owners, describe the campaign once, and an AI call team works the whole list **in parallel** — calling, retrying no-answers, booking service appointments, and handing back a **structured result per customer**.

Built at the **AI BEAVERS × Mollie Founder Hackathon** (Hamburg, June 2026). Concrete wedge: an automotive **recall** team calling 50 owners individually to book service.

> Pitch deck: [`pitch/index.html`](./pitch/index.html) (open in a browser) · [`pitch/Outbound-Autopilot-Rufen-Cara8.pdf`](./pitch/Outbound-Autopilot-Rufen-Cara8.pdf)

---

## What it does

1. **Contacts** — upload `.csv`/`.xlsx` (`name, phone, context, language`); the backend normalizes phones to E.164 and flags invalid rows.
2. **Campaign context builder** — a car-dealership form (campaign type, brand, recall ID, affected models/part, urgency, duration, cost, loaner/pickup…) that composes a rich goal + reason.
3. **AI script** — Claude drafts the agent's system prompt, first message, and a typed **extraction schema**; it asks/confirms **every** field, **one question at a time**.
4. **Voice** — pick from your live **ElevenLabs** voices (with previews).
5. **Settings** — concurrency (capped at 2), retry delay + max attempts, retry-on outcomes.
6. **Review → Test call** — ring your own phone with the exact script/voice before launching.
7. **Launch** — fires a durable **Temporal** workflow that places **real parallel calls** (one ElevenLabs agent per concurrency slot), retries no-answers on a delay, and streams everything to a **live monitor** (status, transcripts, results, aggregates).
8. **Results** — structured per-owner data (booked? preferred date? needs loaner?), **CSV export** (formula-injection sanitized), and **AI insights/summary** generated by a **local Qwen** model.

## Architecture

```
Excel ─► importer ─► CampaignContact rows
Wizard ─► generator (Claude/GPT) ─► script_prompt + first_message + extraction_schema
Launch ─► create N ElevenLabs agents (pool) ─► start Temporal CampaignWorkflow
  CampaignWorkflow ── ≤N concurrent ──► ContactCallWorkflow(contact, agent)
     place_call_activity: ElevenLabs sip-trunk outbound-call (dynamic_vars = {name,context,phone})
        → poll conversation ~1s → publish transcript/status to Redis → classify outcome
        → read analysis.data_collection_results → result
     retry retryable outcomes (durable workflow.sleep) up to max_attempts
Browser ◄─ WS /ws/campaign/{id}/ ◄─ Redis pub/sub   (+ REST polling safety-net)
Insights ◄─ local Qwen (Ollama, OpenAI-compatible)
```

**Why per-agent pool:** ElevenLabs allows **1 concurrent call per agent**, so each campaign creates a small pool of agents (one per concurrency slot) to dial in true parallel.

## Tech stack

- **Backend:** Django 5, DRF, Channels + Daphne (ASGI), Postgres 16, Redis 7
- **Orchestration:** **Temporal** (durable retries, concurrency, fan-out)
- **Voice:** **ElevenLabs** Agents (STT → `claude-haiku-4-5` → TTS) over a **Telnyx** SIP trunk
- **Script generation:** Claude / GPT (`ORCHESTRATOR_PROVIDER`)
- **Insights:** **local Qwen** (`qwen2.5` via Ollama)
- **Frontend:** React 19 + Vite + Tailwind 4 (design tokens, black + orange)

## Run it

```bash
# backend + infra (db · redis · temporal(+UI :8233) · web :8000 · temporal-worker)
docker compose up --build
docker compose exec web python manage.py migrate
docker compose exec web python manage.py seed_demo --reset      # demo campaigns

# frontend
cd frontend && npm install && npm run dev -- --host --port 3001   # → http://localhost:3001

# AI insights (local Qwen) — optional
brew install --cask ollama && open -a Ollama
ollama pull qwen2.5:32b        # or qwen2.5:3b for speed
```

**Real phone calls** need the Telnyx → ElevenLabs SIP import done and `ELEVEN_AGENT_PHONE_NUMBER_ID` set in `.env` (see [`docs/import_number.md`](./docs/import_number.md)). Without it, drive the live monitor with `python manage.py simulate_run <campaign_id>`.

**One-shot test call:** `docker compose exec web python manage.py fire_test_call +49…`

## Env (`.env`, gitignored — template `.env.example`)

`ELEVENLABS_API_KEY`, `ELEVEN_AGENT_PHONE_NUMBER_ID`, `TELNYX_API_KEY`, `TELNYX_FROM_NUMBER`, `ANTHROPIC_API_KEY` / `OPENAI_API_KEY` + `ORCHESTRATOR_PROVIDER`, `TEMPORAL_HOST`/`TEMPORAL_NAMESPACE`, `QWEN_BASE_URL`/`QWEN_MODEL`/`QWEN_API_KEY`.

## Notes

- **Auth + billing are mocked** for the hackathon (no users/tenancy); endpoints are open and local-only. Production would add auth + per-tenant scoping + rate limiting.
- Concurrency is capped at **2** (min of the ElevenLabs plan and the Telnyx channel limit).
